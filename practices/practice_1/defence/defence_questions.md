## Вопросы по теории SNMP

### 1. Что такое SNMP и для каких целей он используется?
**Ответ:**
SNMP (Simple Network Management Protocol) - это протокол управления сетью, предназначенный для:
- Мониторинга сетевых устройств (серверы, маршрутизаторы, коммутаторы)
- Сбора информации о состоянии системы (загрузка CPU, память, диски)
- Получения уведомлений о событиях (traps)
- Удаленного управления параметрами устройств

Работает на прикладном уровне модели OSI, использует порты UDP 161 (агент) и 162 (менеджер).

### 2. Какие версии SNMP существуют и в чем их основные отличия?
**Ответ:**
- **SNMPv1** - первая версия, простая аутентификация через community strings, слабая безопасность
- **SNMPv2c** - улучшенная производительность, новые типы данных, но та же community-аутентификация
- **SNMPv3** - современная версия с шифрованием, аутентификацией (MD5/SHA) и контролем доступа (USM)

В работе использовалась версия 2c (флаг `-v 2c`).

### 3. Что такое OID (Object Identifier) и какова его структура?
**Ответ:**
OID - это уникальный идентификатор объекта в дереве MIB. Структура:
- Иерархическое дерево, разделенное точками
- Каждый узел имеет числовой и текстовый идентификатор
- Корень: iso (1) → org (3) → dod (6) → internet (1)

Пример: `.1.3.6.1.4.1.2021.4.3.0`
- `.1` - iso
- `.3` - org
- `.6` - dod
- `.1` - internet
- `.4` - private
- `.1` - enterprises
- `.2021` - ucdavis (UCD-SNMP-MIB)
- `.4` - memory
- `.3` - memTotalSwap
- `.0` - скалярный объект

### 4. Что такое MIB (Management Information Base) и для чего он нужен?
**Ответ:**
MIB - это база управляющей информации, которая:
- Определяет структуру управляемых объектов
- Содержит текстовые имена для OID
- Описывает типы данных, доступные операции (read/write)
- Позволяет интерпретировать числовые OID

Стандартные MIB: MIB-II (RFC 1213), UCD-SNMP-MIB, HOST-RESOURCES-MIB.

### 5. В чем разница между скалярными объектами и таблицами в SNMP?
**Ответ:**
- **Скалярные объекты** - одиночные значения, имеют суффикс `.0` в OID (например, `.1.3.6.1.4.1.2021.4.3.0` - total swap)
- **Таблицы** - наборы данных с несколькими строками, имеют индексы (например, `.1.3.6.1.4.1.2021.9.1.2.1` - первый диск, `.1.3.6.1.4.1.2021.9.1.2.2` - второй диск)

### 6. Какие типы данных используются в SNMP?
**Ответ:**
- INTEGER - целые числа (статистика, счетчики)
- OCTET STRING - строки (имена, описания)
- OBJECT IDENTIFIER - ссылки на другие OID
- NULL - пустое значение
- Counter32/Counter64 - счетчики (не сбрасываются)
- Gauge32 - датчики (могут увеличиваться и уменьшаться)
- TimeTicks - время в сотых долях секунды
- IpAddress - IP-адреса

### 7. Что такое community string и как она используется в SNMPv2c?
**Ответ:**
Community string - это пароль или ключ доступа:
- Выполняет роль базовой аутентификации
- Передается в открытом виде (не шифруется)
- Действует как пароль для доступа к агенту
- Может иметь разные права: read-only (только чтение), read-write (чтение и запись)

В работе используется community: `commavt-341-9`

## Вопросы по настройке SNMP

### 8. Какие пакеты необходимо установить для работы SNMP в CentOS 6?
**Ответ:**
```bash
yum install net-snmp-libs net-snmp net-snmp-utils
```

### 9. Объясните назначение каждой строки в конфигурации snmpd.conf:
**Ответ:**
```bash
com2sec local 127.0.0.1 commavt-341-9
```
- Создает security name "local" для IP 127.0.0.1 с community "commavt-341-9"

```bash
group avt341-9 any local
```
- Группирует security name "local" в группу "avt341-9" с контекстом "any"

```bash
view all included .1
```
- Создает view (представление) "all", включающее всю ветку OID (.1)

```bash
access avt341-9 "" any noauth 0 all none none
```
- Настраивает доступ для группы: noauth (без аутентификации), 0 (precise), all (view для чтения)

```bash
syslocation Lab_403
```
- Устанавливает местоположение системы

```bash
syscontact IS-21, Ivanov Petrov
```
- Указывает контактную информацию

```bash
includeAllDisks 10%
```
- Включает мониторинг всех дисков, где свободно менее 10%

### 10. Для чего нужна директива `includeAllDisks 10%`?
**Ответ:**
Эта директива:
- Включает автоматический мониторинг всех смонтированных дисковых разделов
- Добавляет их в таблицу dskTable (OID .1.3.6.1.4.1.2021.9.1)
- Параметр 10% означает, что в таблицу включаются разделы, где свободно менее 10%, иначе они не отслеживаются
- Без этой директивы таблица дисков может быть пустой

### 11. Какие команды используются для управления сервисом snmpd?
**Ответ:**
```bash
service snmpd start   # запуск сервиса
service snmpd stop    # остановка
service snmpd restart # перезапуск
service snmpd status  # проверка статуса
chkconfig snmpd on    # добавление в автозагрузку
```

### 12. Как проверить корректность конфигурации snmpd.conf?
**Ответ:**
```bash
snmpd -f -Le -DALL 2>&1 | head -20
```
- `-f` - запуск в foreground (не как демон)
- `-Le` - логирование в stderr
- `-DALL` - включение всех отладочных сообщений

Или:
```bash
snmpwalk -v 2c -c community 127.0.0.1 .1
```
Если возвращаются данные - конфигурация верна.

## Вопросы по утилитам SNMP

### 13. В чем разница между snmpget и snmpwalk?
**Ответ:**
- **snmpget** - получает значение одного конкретного OID (требует точный OID с индексом)
- **snmpwalk** - рекурсивно обходит все объекты в указанной ветке (использует GETNEXT)

Пример:
```bash
snmpget -v 2c -c community 127.0.0.1 .1.3.6.1.4.1.2021.4.3.0  # один объект
snmpwalk -v 2c -c community 127.0.0.1 .1.3.6.1.4.1.2021.9.1   # вся ветка дисков
```

### 14. Как использовать snmpwalk для получения информации о дисках?
**Ответ:**
```bash
# Получить всю ветку дисков
snmpwalk -v 2c -c commavt-341-9 127.0.0.1 .1.3.6.1.4.1.2021.9.1

# Получить только точки монтирования
snmpwalk -v 2c -c commavt-341-9 127.0.0.1 .1.3.6.1.4.1.2021.9.1.2

# Получить данные для конкретного диска (индекс 1)
snmpwalk -v 2c -c commavt-341-9 127.0.0.1 .1.3.6.1.4.1.2021.9.1.2.1
snmpwalk -v 2c -c commavt-341-9 127.0.0.1 .1.3.6.1.4.1.2021.9.1.6.1
```

### 15. Что означает флаг `-v 2c` в командах snmpwalk/snmpget?
**Ответ:**
- `-v` - указывает версию протокола SNMP
- `2c` - версия SNMPv2c (вторая версия с community-аутентификацией)
- Альтернативы: `-v 1` (SNMPv1), `-v 3` (SNMPv3)

### 16. Какие OID используются для получения информации о swap и дисках?
**Ответ:**

**Swap:**
- Total Swap: `.1.3.6.1.4.1.2021.4.3.0`
- Available Swap: `.1.3.6.1.4.1.2021.4.4.0`

**Диски:**
- Точка монтирования: `.1.3.6.1.4.1.2021.9.1.2.{index}`
- Устройство: `.1.3.6.1.4.1.2021.9.1.3.{index}`
- Всего (kB): `.1.3.6.1.4.1.2021.9.1.6.{index}`
- Занято (kB): `.1.3.6.1.4.1.2021.9.1.8.{index}`
- Доступно (kB): `.1.3.6.1.4.1.2021.9.1.7.{index}`
- Процент использования: `.1.3.6.1.4.1.2021.9.1.9.{index}`

### 18. Как вычислить использованный swap, если в SNMP нет прямого OID?
**Ответ:**
Использованный swap вычисляется по формуле:
```
Used Swap = Total Swap - Available Swap
```
Где:
- Total Swap: `.1.3.6.1.4.1.2021.4.3.0`
- Available Swap: `.1.3.6.1.4.1.2021.4.4.0`

### 19. Как в awk извлечь числовые значения из вывода snmpwalk?
**Ответ:**
Несколько способов:

1. **Через разделитель полей:**
```bash
snmpwalk ... | awk -F'[.:: ]+' '{print $6}'
```

2. **Через регулярное выражение:**
```bash
snmpwalk ... | awk '{print $NF}'  # последнее поле
snmpwalk ... | grep -oP 'INTEGER: \K[0-9]+'
```

3. **Через gsub:**
```bash
awk '{gsub(/[^0-9]/, "", $4); print $4}'
```

### 20. Как в awk обработать табличные данные и сгруппировать по индексам?
**Ответ:**
Используются ассоциативные массивы:
```bash
awk -F' = ' '
{
    split($1, parts, ".")
    param = parts[1]
    idx = parts[2]
    value = $2
    if (param == "dskPath") path[idx] = value
    if (param == "dskDevice") device[idx] = value
    if (param == "dskTotal") total[idx] = value
}
END {
    for (i in path) {
        print path[i], device[i], total[i]
    }
}'
```

### 22. Какие данные содержит dskPercentNode и для чего он нужен?

## Вопросы по обработке данных

### 23. Как в awk работает разделитель полей `-F'[.:: ]+'`?
**Ответ:**
- `-F` - опция установки разделителя
- `'[.:: ]+'` - регулярное выражение:
  - `[.:: ]` - класс символов: точка, двоеточие, пробел
  - `+` - квантификатор "один или более раз"
- Строка разбивается по любому количеству этих символов
- Пример: `"UCD-SNMP-MIB::memTotalSwap.0"` → поля: "UCD-SNMP-MIB", "memTotalSwap", "0"

### 24. Что означают символы `^` и `$` в регулярных выражениях awk?
**Ответ:**
- `^` - якорь начала строки (matches start of line)
- `$` - якорь конца строки (matches end of line)

Примеры:
- `/^STRING:/` - строки, начинающиеся с "STRING:"
- `/\.0$/` - строки, оканчивающиеся на ".0"
- `/^.*::/` - от начала до "::"

### 25. Как работает функция gsub в awk?
**Ответ:**
`gsub(regex, replacement, string)` - глобальная замена:
- `regex` - регулярное выражение для поиска
- `replacement` - строка для замены
- `string` - целевая строка (если не указана, используется `$0`)

Пример:
```awk
gsub(/^.*::/, "", $1)  # удаляет всё до "::" в $1
gsub(/^STRING: /, "", value)  # удаляет "STRING: " в начале
```

### 26. Как в awk извлечь индекс из OID?
**Ответ:**
Несколько способов:

1. **Через split:**
```awk
split($1, parts, ".")
idx = parts[length(parts)]  # последняя часть
```

2. **Через match с регулярным выражением:**
```awk
match($1, /\.([0-9]+)$/, arr)
idx = arr[1]  # последнее число
```

3. **Через gsub:**
```awk
gsub(/.*\./, "", $1)  # удаляет всё до последней точки
idx = $1
```

### 27. Почему в скрипте используется `>>` вместо `>`?
**Ответ:**
- `>` - перенаправление с перезаписью файла (создает новый или очищает существующий)
- `>>` - перенаправление с добавлением в конец файла (append)

Использование `>>` позволяет:
- Последовательно добавлять результаты разных команд
- Формировать структурированный отчет
- Не потерять данные при многократном выполнении

## Дополнительные вопросы

### 33. Какие проблемы могут возникнуть при настройке SNMP и как их решить?
**Ответ:**

| Проблема | Решение |
|----------|---------|
| Сервис не запускается | Проверить синтаксис snmpd.conf, посмотреть логи `/var/log/messages` |
| Пустой вывод snmpwalk | Проверить community, настройки доступа (com2sec, group, access) |
| Нет данных о дисках | Добавить `includeAllDisks` в конфигурацию |
| Ошибка "No response" | Проверить сетевые настройки, файервол (порт 161) |
| Неправильные значения | Проверить OID, убедиться что используется верный индекс |

### 34. Как проверить, что SNMP агент слушает порт?
**Ответ:**
```bash
# Проверка через netstat
netstat -tulpn | grep 161
# или
ss -tulpn | grep 161

# Проверка через nmap
nmap -p 161 127.0.0.1

# Проверка через lsof
lsof -i :161
```

### 35. Какие права доступа должны быть у файла snmpd.conf?
**Ответ:**
```bash
-rw------- 1 root root  /etc/snmp/snmpd.conf
```
- Владелец: root
- Права: 600 (только root может читать и писать)
- Это важно для безопасности, так как файл содержит community strings

### 36. Как настроить SNMP для мониторинга с удаленного хоста?
**Ответ:**
В файле `/etc/snmp/snmpd.conf`:
```bash
# Добавить строку для доступа с конкретного IP
com2sec remote 192.168.1.0/24 commavt-341-9
# Или для всех
com2sec remote default commavt-341-9

# Добавить в группу
group avt341-9 any remote

# В директиве access указать контекст
access avt341-9 "" any noauth 0 all none none
```
И открыть порт 161 в файерволе:
```bash
iptables -A INPUT -p udp --dport 161 -j ACCEPT
service iptables save
```

### 38. Что означает параметр `noauth` в директиве access?
**Ответ:**
`noauth` - уровень безопасности:
- `noauth` - без аутентификации (SNMPv1/v2c)
- `auth` - только аутентификация (SNMPv3)
- `priv` - аутентификация и шифрование (SNMPv3)

Для SNMPv2c всегда используется `noauth`.

### 39. Какие существуют способы форматирования вывода в awk?
**Ответ:**
1. **printf** - форматированный вывод:
   ```awk
   printf "%-20s | %-10d\n", name, value
   ```

2. **print** - простой вывод с пробелами:
   ```awk
   print $1, $2, $3
   ```

3. **Конкатенация**:
   ```awk
   print $1 ": " $2
   ```

4. **OFMT** - формат чисел:
   ```awk
   OFMT="%.2f"
   ```
