			Практическая работа #1

	Тема: Мониторинг состояния OS GNU/Linux при помощи протокола SNMP

Исходные данные:
	Операционная система CentOS Linux серсии 6.x с административными правами


Задание:
	1) Установить пакеты 
		net-snmp-libs,
		net-snmp,
		net-snmp-utils
	
	   yum install {имена пакетов через пробел}

	2) Настроить доступ и community для snmpd в файле /etc/snmp/snmpd.conf

	   правила составления имен:
	   variant = название группы + номер бригады
	   community name = comm + variant

	   добавить строки:
	   com2sec	local	127.0.0.1	{community name}
	   group	{variant}	any	local
	   view	all	included	.1
	   access {variant} ""	any noauth	0	all	none	none

	   Изменить строки:
	   syslocation {место выполнения работы}
	   syscontact {группа, студенты}

	3) Запустить сервис snmpd и проверить его работу
	   
	   service snmpd start
	   snmpwalk -v 2c -c {community name} 127.0.0.1

	4) Извлечь при помощи snmp только ту информацию, которая указана в варианте индивидуального задания,
	   результат сохранить в файле /root/{variant}.log
           Не допускается прямое или косвенное использование утилиты snmptable, результат должен быть оформлен
           в виде "параметр: значение", имя параметра без snmp префикса, если подразумевается несколько значений,
           результат должен быть оформлен в виде таблицы

	5) Сохранить файл из п.4 и /etc/snmp/snmpd.conf на локальный компьютер при помощи утилиты pscp

Варианты заданий:
9) Информация о дисковых устройствах, файле подкачки (swap) (всего, занято, свободно)
 


Содержание отчета:
1) Список команд, использованных при выполнении индивидуального задания
2) Файл /root/{variant}.log
3) Файл /etc/snmp/snmpd.conf


Полезные сведения:

{параметр1} - обязательный параметр (от этом говорят фигурные скобки), сами скобки не указываются
[параметр] - необязательный параметр (от этом говорят квадратные скобки), сами скобки не указываются

Полезная информация:
https://net-snmp.sourceforge.io/

Все параметры, которые можно получить от устройства по протоколу SNMP, можно увидеть с помощью команды:

snmpwalk -v 2c -c {community name} -v2c 172.16.8.X

    где 172.16.8.Х - ip адрес вашего сервера в соответствии с вариантом

Чтобы получить значение конкретной ветки или параметра, нужно добавить его OID. Например, чтобы получить время работы системы:

snmpwalk -v 2c -c {community name} 172.16.8.X .1.3.6.1.2.1.1.3.0
