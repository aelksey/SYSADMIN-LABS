Задание:
    1) Создать файл отчета /root/sysadm-lab3-{группа}-{вариант}.log, выполнив команду
		 echo $(date) > /root/sysadm-lab3-{группа}-{вариант}.log

	Все команды и их результат - записывать в файл отчета.

    2) Подключение дисков по iscsi
	
	Установить пакет open-iscsi при помощи утилиты apt
 
	Изменить файл /etc/iscsi/initiatorname.iscsi: 
	InitiatorName=iqn.2025-04.ru.nstu.cs.vt:{группа}v{бригада}.initiator

	Где {группа}v{бригада} - название группы и номер бригады латинскими буквами, 
		v - это буква, {} - скобки, в результат не включаются

	Изменить файл /etc/iscsi/iscsid.conf, модифицировав следующие строки:
	
	    node.session.auth.authmethod = CHAP	    
	    node.session.auth.username = username
	    node.session.auth.password = password

	Где username и password - п.2 исходных данных

	Не забыть выполнить перезапуск сервисов iscsid и open-iscsi:
	systemctl restart iscsid open-iscsi 

	Выполнить команду iscsiadm -m discovery -t sendtargets -p {портал},  где {портал} - это адрес портала п.2
	исходных данных. Если предыдущие шаги выполенны правильно, то должен появиться список, сожержащий множество
	тагретов вида iqn.2025-04.ru.nstu.cs.vt:..., среди которых необходимо найти свой по номеру группы и бригады.
	Результат выполнения этой команды необходимо записать в файл отчета.

	Сохранить содержимое файла /proc/partitions в файл /root/before.txt

	Выполнить команду iscsiadm --mode node --targetname {таргет бригады} --login, результаты вывода
	этой команды и команды iscsiadm -m session -o show записать в файл отчета. 

	Выполнить команду diff /root/before.txt /proc/partitions и сохранить результат ее работы в файл отчета.
	Это позволит определить, какие новые устройства и с какими именами были добавлены в систему.
	Файл /root/before.txt больше не нужен, его можно удалить.

	Если предыдущие шаги выполнены правильно, то в распоряжении группы теперь имеется два блочных устройства
	по 512 МБ каждое и можно переходить к следующему шагу.

    3) Создание программного raid-массива
	
	С помощью утилиты parted создать на полученных в п.2 задания блочных устройствах по одному первичному
	разделу с типом fd (Linux raid)

	Сохранить в файл отчета результат работы parted /dev/{имя блочного устройства} print для каждого из устройств
	
	Установить пакет mdadm (software RAID) и выполнить команду mdadm --detail --scan и определить доступное имя
	для raid-массива. Имена имеют вид /dev/mdX, где X - номер raid-массива, начиная с 0, если список пуст, 
	то можно использовать 0, иначе необходимо найти не занятое имя.

	Создать программный raid-массив 1 уровня (зеркало) при помощи команды
	 mdadm --create --verbose /dev/{имя raid-масива} --level=1 --raid-devices=2 /dev/{раздел1} /dev/{раздел2}

	Где {раздел1} и {раздел2} - это разделы, созданные на предыдущем шаге при помощи parted.

	Сохранить содержимое файла /proc/mdstat и результат вывода команды mdadm --detail --scan в файл отчета

    4) Использование LVM

	Разметить полученный в п.3 задания raild-массив как физический том lvm при помощи команды pvcreate

	При помощи команды vgcreate создать группу томов с именем vg-{группа}-{вариант} и включить в нее
	свой физический том.

	При помощи команды lvcreate создать логический том с произвольным именем (ЛТсПИ) в группе томов vg-{группа}-{вариант}
	размером 70% от всего доступного места в группе томов.
	
	Создать на /dev/vg-{группа}-{вариант}/{ЛТсПИ} файловую систему ext4 при помощи команды mkfs

	Сохранить результат работы команд pvdisplay, vgdisplay, lvdisplay и blkid /dev/vg-{группа}-{вариант}/{ЛТсПИ}
	в файл отчета.

	Создать в каталоге /mnt подкаталог для монтирования созданной файловой системе, названный по принципу
	/mnt/{ЛТсПИ}_base (точку монтирования) и смонтировать его при помощи команды mount.

	При помощи утилиты dd создать в каталоге /mnt/{ЛТсПИ}_base файл test.bin размером 100МБ из источника /dev/random
	и вычислить для полученного файла контрольную сумму при помощи программы md5sum, сохранить это значение в файле журнала

	При помощи команды lvcreate cоздать для {ЛТсПИ} снимок (snapshot) размером со все свободное место на группе томов и
 	именем {ЛТсПИ}_snap, создать для него точку монтирования /mnt/{ЛТсПИ}_snap и выполнить монтирование

	При помощи утилиты dd заменить в каталоге /mnt/{ЛТсПИ}_base файл test.bin размером 100МБ из источника /dev/random,
	вычислить контрольные суммы md5 для файлов /mnt/{ЛТсПИ}_base/test.bin и /mnt/{ЛТсПИ}_snap/test.bin и сохранить 
	эти значения в файле журнала.

	Сохранить в файле отчета результат работы lvdisplay, размонтировать каталог /mnt/{ЛТсПИ}_snap при помощи umount 
	и удалить снимок логического тома {ЛТсПИ}_snap при помощи команды lvremove
