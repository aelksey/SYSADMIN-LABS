# Вопросы для защиты лабораторной работы №3

## Вопросы по iSCSI

### 1. Что такое iSCSI и для чего он используется?
**Ответ:** iSCSI (Internet Small Computer System Interface) — это протокол для передачи команд SCSI по TCP/IP сетям. Он позволяет организовать доступ к блочным устройствам (дискам, LUN) по сети, делая удаленные диски видимыми для операционной системы как локальные. Используется для создания сетей хранения данных (SAN) без необходимости в дорогом оборудовании Fibre Channel.

### 2. Что такое InitiatorName и как он формируется в вашей работе?
**Ответ:** InitiatorName — это уникальный идентификатор iSCSI-инициатора (клиента) в формате IQN (iSCSI Qualified Name). В работе он был сформирован по шаблону:
```
iqn.2025-04.ru.nstu.cs.vt:avt341.v9.initiator
```
Где:
- `iqn` — тип идентификатора
- `2025-04` — год и месяц регистрации домена
- `ru.nstu.cs.vt` — обратное доменное имя
- `avt341.v9` — группа и бригада
- `initiator` — указание, что это инициатор

### 3. Для чего нужна CHAP-аутентификация в iSCSI?
**Ответ:** CHAP (Challenge-Handshake Authentication Protocol) обеспечивает безопасность подключения. Сервер отправляет клиенту "вызов" (случайную строку), клиент хэширует его с паролем и отправляет обратно. Сервер проверяет результат. Это предотвращает несанкционированный доступ к LUN, так как пароль не передается в открытом виде.

### 4. В чем разница между discovery и login в iSCSI?
**Ответ:**
- **Discovery** — процесс поиска доступных таргетов на iSCSI-сервере. Клиент узнает, какие LUN ему разрешены.
- **Login** — процесс установки сессии с конкретным таргетом. После успешного логина блочное устройство становится доступным в системе.

### 5. Что означает ошибка "No records found" при выполнении iscsiadm --login?
**Ответ:** Эта ошибка возникает, когда в локальной базе `/var/lib/iscsi/nodes/` нет записи о таргете. Обычно это происходит, если:
- Discovery не был выполнен перед логином
- Имя таргета указано неверно (например, с IP-адресом)
- Discovery был выполнен, но с другими параметрами аутентификации

## Вопросы по RAID

### 6. Что такое RAID и какие уровни вы знаете?
**Ответ:** RAID (Redundant Array of Independent Disks) — технология объединения дисков в массив для повышения производительности и/или отказоустойчивости.

Основные уровни:
- **RAID 0** (Striping) — чередование данных. Повышает скорость, но нет отказоустойчивости.
- **RAID 1** (Mirroring) — зеркалирование. Данные дублируются на двух дисках. При выходе одного диска данные сохраняются.
- **RAID 5** — чередование с четностью. Требует минимум 3 диска.
- **RAID 6** — двойная четность. Выдерживает отказ 2 дисков.
- **RAID 10** — комбинация RAID 1+0.

### 7. Какой уровень RAID использовался в работе и почему?
**Ответ:** Использовался **RAID 1 (зеркало)**. Он выбран потому, что:
- Обеспечивает отказоустойчивость — при выходе одного из двух дисков данные сохраняются
- Прост в создании и понимании
- Требует минимум 2 диска (как в задании)
- Размер массива равен размеру меньшего диска (512 МБ)

### 8. Как посмотреть статус RAID-массива в Linux?
**Ответ:** Основные команды:
```bash
cat /proc/mdstat           # Краткий статус всех массивов
mdadm --detail /dev/md0    # Детальная информация о конкретном массиве
mdadm --detail --scan      # Сканирование и вывод всех массивов
```

### 9. Что означает тип раздела "fd" (Linux raid) в parted?
**Ответ:** Тип раздела "fd" (или 0xFD в hex) — это специальный идентификатор, указывающий ядру Linux, что этот раздел является частью программного RAID-массива. Это позволяет системе автоматически распознавать и собирать RAID-массивы при загрузке.

## Вопросы по LVM

### 10. Что такое LVM и какие компоненты входят в его состав?
**Ответ:** LVM (Logical Volume Manager) — система управления дисковым пространством, позволяющая гибко перераспределять место.

Компоненты:
- **PV (Physical Volume)** — физический том (раздел или диск)
- **VG (Volume Group)** — группа томов, объединяющая несколько PV
- **LV (Logical Volume)** — логический том, создается внутри VG, используется как блочное устройство

### 11. В чем преимущества LVM перед обычной разметкой диска?
**Ответ:**
- **Гибкость** — можно изменять размер томов "на лету"
- **Снимки (snapshots)** — мгновенные копии состояния
- **Объединение дисков** — один VG может включать несколько физических дисков
- **Thin provisioning** — выделение места по мере необходимости
- **Миграция** — перемещение данных между физическими томами без остановки работы

### 12. Что такое snapshot в LVM и для чего он используется?
**Ответ:** Snapshot (снимок) — это "замороженная" копия логического тома в определенный момент времени. Работает по принципу "copy-on-write": при изменении данных в исходном томе, их старая версия копируется в снимок.

**Применение:**
- Резервное копирование работающей системы
- Тестирование обновлений
- Откат к предыдущему состоянию
- Анализ данных без изменения оригинала

### 13. Почему при замене файла в исходной ФС, в снимке осталась старая контрольная сумма?
**Ответ:** Это и есть принцип работы снимка. Снимок фиксирует состояние файловой системы на момент своего создания. Когда вы заменили файл `test.bin` в исходной ФС, LVM:
1. Скопировал старую версию файла в область снимка
2. Затем записал новую версию в исходный том

Поэтому:
- Исходный том содержит **новый** файл → новая md5 сумма
- Снимок содержит **старый** файл → старая md5 сумма

### 14. Какие команды используются для просмотра информации о LVM-томах?
**Ответ:**
```bash
pvdisplay   # Информация о Physical Volumes
vgdisplay   # Информация о Volume Groups
lvdisplay   # Информация о Logical Volumes
pvs, vgs, lvs  # Краткий вывод в табличном виде
blkid       # UUID и тип ФС для блочных устройств
```

### 15. Как создать снимок и как его удалить?
**Ответ:**
**Создание:**
```bash
lvcreate -l 100%FREE -s -n snapshot_name /dev/vg_name/lv_name
# -l 100%FREE — использовать все свободное место в VG
# -s — создать снимок (snapshot)
# -n — имя снимка
```

**Удаление:**
```bash
umount /mount_point           # Размонтировать, если смонтирован
lvremove -f /dev/vg_name/snapshot_name
```

## Общие вопросы

### 16. Как изменится размер RAID-массива, если один из дисков имеет меньший объем?
**Ответ:** В RAID 1 размер массива определяется размером **меньшего** диска. Например, если первый диск 500 ГБ, а второй 1 ТБ, то массив будет 500 ГБ. Остальное пространство большего диска останется неиспользованным.

### 17. Почему для команды dd использовался /dev/urandom вместо /dev/random?
**Ответ:** `/dev/random` блокируется, если в системе недостаточно энтропии для генерации случайных чисел. `/dev/urandom` — неблокирующий аналог, который использует криптостойкий генератор псевдослучайных чисел. Для тестовых целей (не для криптографии) `/dev/urandom` подходит идеально и не вызывает зависаний.

### 18. Какая команда показывает, какие блочные устройства появились после подключения iSCSI?
**Ответ:**
```bash
cat /proc/partitions        # Список всех блочных устройств
lsblk                       # Древовидный вывод блочных устройств
dmesg | tail                # Последние сообщения ядра
diff before.txt /proc/partitions  # Сравнение "до" и "после"
```

### 19. Что такое extent в LVM?
**Ответ:** Extent — минимальная единица выделения места в LVM.
- **PE (Physical Extent)** — размер блока на физическом томе
- **LE (Logical Extent)** — размер блока на логическом томе
По умолчанию размер одного extent = 4 МБ. Все операции выделения места в LVM кратны размеру extent.

### 20. Какой файл нужно отредактировать, чтобы iSCSI-инициатор автоматически подключался при загрузке?
**Ответ:** Файл `/etc/iscsi/iscsid.conf`. Параметр `node.startup = automatic` (вместо `manual`) обеспечивает автоматическое подключение к таргетам при загрузке системы. Также настройки сохраняются в `/etc/iscsi/nodes/` и `iscsiadm` использует их для восстановления сессий.
