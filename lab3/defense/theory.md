## 1. Технология iSCSI

### 1.1 Что такое iSCSI?

**iSCSI** (Internet Small Computer System Interface) — это протокол транспортного уровня, который позволяет передавать команды протокола SCSI поверх TCP/IP сетей.

**Простая аналогия:** Обычный SCSI — это когда диск подключен напрямую к материнской плате. iSCSI — это тот же диск, но подключенный через сеть, как если бы вы удлинили SCSI-кабель на сотни метров.

### 1.2 Архитектура iSCSI

```
┌─────────────────┐                    ┌─────────────────┐
│   iSCSI Client  │                    │   iSCSI Server  │
│  (Initiator)    │◄───────TCP/IP──────►│   (Target)      │
│                 │                    │                 │
│  - open-iscsi   │                    │  - LUN (диск)   │
│  - ищет target  │                    │  - целевое      │
│  - подключается │                    │    устройство   │
└─────────────────┘                    └─────────────────┘
```

**Ключевые понятия:**

| Термин | Описание |
|--------|----------|
| **Initiator** | Клиент iSCSI (ваш сервер), который подключается к удаленным дискам |
| **Target** | Сервер iSCSI, который предоставляет доступ к LUN |
| **LUN** | Logical Unit Number — блочное устройство (виртуальный диск) |
| **IQN** | iSCSI Qualified Name — уникальное имя в формате `iqn.гггг-мм.домен:имя` |
| **Portal** | IP-адрес и порт (обычно 3260) iSCSI-сервера |
| **CHAP** | Протокол аутентификации |

### 1.3 Формат IQN (iSCSI Qualified Name)

```
iqn.2025-04.ru.nstu.cs.vt:avt341.v9.initiator
│   │       │            │        │
│   │       │            │        └─── конкретное имя (группа.бригада.роль)
│   │       │            └─── поддомен (лаборатория vt)
│   │       └─── домен в обратном порядке
│   └─── год и месяц регистрации
└─── тип идентификатора (iSCSI Qualified Name)
```

### 1.4 Процесс подключения iSCSI

```
1. Discovery
   Initiator ──"Какие target'ы у тебя есть?"──► Target
   Initiator ◄──"Вот список разрешенных target'ов"── Target

2. Login
   Initiator ──"Подключаюсь к target X"──► Target
   Initiator ◄──"Успешно, вот твой LUN"── Target

3. Использование
   Initiator ◄─────Чтение/Запись────► Target (LUN появляется как /dev/sdX)
```

### 1.5 CHAP-аутентификация

**CHAP** (Challenge-Handshake Authentication Protocol) — защищенный метод аутентификации:

```
1. Initiator ──"Я пользователь username"──► Target
2. Target ──"Докажи. Вот случайное число (challenge)"──► Initiator
3. Initiator ──"Вот MD5(challenge + password)"──► Target
4. Target ──"Доступ разрешен/запрещен"──► Initiator
```

**Преимущество:** Пароль не передается по сети в открытом виде.

### 1.6 Конфигурационные файлы open-iscsi

| Файл | Назначение |
|------|------------|
| `/etc/iscsi/initiatorname.iscsi` | Уникальное имя инициатора |
| `/etc/iscsi/iscsid.conf` | Глобальные настройки (CHAP, таймауты) |
| `/var/lib/iscsi/nodes/` | База данных известных таргетов |
| `/etc/iscsi/send_targets/` | Настройки discovery |

## 2. Программный RAID (mdadm)

### 2.1 Что такое RAID?

**RAID** (Redundant Array of Independent Disks) — технология объединения нескольких физических дисков в один логический массив для повышения производительности и/или отказоустойчивости.

### 2.2 Основные уровни RAID

```
RAID 0 (Striping - чередование)
┌────────┬────────┐
│ Диск 1 │ Диск 2 │
├────────┼────────┤
│ Данные │ Данные │
│ Блок 1 │ Блок 2 │
│ Блок 3 │ Блок 4 │
└────────┴────────┘
+ Производительность (чтение/запись с двух дисков)
- Нет отказоустойчивости (выход одного диска — потеря всех данных)

RAID 1 (Mirroring - зеркалирование)
┌────────┬────────┐
│ Диск 1 │ Диск 2 │
├────────┼────────┤
│ Данные │ Данные │
│ Блок 1 │ Блок 1 │  ← копия
│ Блок 2 │ Блок 2 │  ← копия
└────────┴────────┘
+ Отказоустойчивость (выход одного диска — данные целы)
- Полезный объем = размер меньшего диска

RAID 5 (Striping с четностью)
┌────────┬────────┬────────┐
│ Диск 1 │ Диск 2 │ Диск 3 │
├────────┼────────┼────────┤
│ Блок 1 │ Блок 2 │ Parity │
│ Parity │ Блок 3 │ Блок 4 │
└────────┴────────┴────────┘
+ Эффективно использует место (N-1 дисков)
+ Выдерживает отказ 1 диска

RAID 10 (RAID 1 + 0)
┌─────────────────────────┐
│    RAID 0 (Striping)    │
├────────────┬────────────┤
│  RAID 1    │  RAID 1    │
├──────┬─────┼──────┬─────┤
│Диск 1│Диск2│Диск3│Диск4│
└──────┴─────┴──────┴─────┘
+ Высокая производительность + отказоустойчивость
- Требует минимум 4 диска
```

### 2.3 Формулы расчета

| RAID | Полезный объем | Отказоустойчивость | Мин. дисков |
|------|---------------|-------------------|-------------|
| 0 | Сумма всех дисков | 0 дисков | 2 |
| 1 | Размер меньшего диска | 1 диск | 2 |
| 5 | (N-1) × мин.диск | 1 диск | 3 |
| 6 | (N-2) × мин.диск | 2 диска | 4 |
| 10 | (N/2) × мин.диск | 1-2 диска | 4 |

### 2.4 Типы разделов для RAID

В `parted` или `fdisk` для RAID-разделов используется специальный тип:

| Утилита | Код типа | Назначение |
|---------|----------|------------|
| parted | `fd` или `raid` | Linux RAID |
| fdisk | `fd` (Linux RAID) | Linux RAID |
| gdisk | `FD00` | Linux RAID |

Этот код сообщает ядру, что раздел является частью RAID-массива.

### 2.5 Структура mdadm

```bash
# Файл конфигурации
/etc/mdadm/mdadm.conf     # Список массивов для автоматической сборки

# Устройства
/dev/md0, /dev/md1, ...   # Имена RAID-массивов

# Информация о состоянии
/proc/mdstat              # Текущее состояние всех массивов
```

## 3. LVM (Logical Volume Manager)

### 3.1 Что такое LVM?

**LVM** (Logical Volume Manager) — это менеджер томов, который добавляет уровень абстракции между физическими дисками и файловыми системами.

### 3.2 Иерархия LVM

```
Физические диски
     ↓
┌─────────────────────────────────────────────┐
│  /dev/sda    /dev/sdb    /dev/sdc           │
│     ↓           ↓           ↓               │
│  PV (Physical Volume)                       │
│  (размеченный диск для LVM)                  │
└─────────────────────────────────────────────┘
     ↓
┌─────────────────────────────────────────────┐
│           VG (Volume Group)                 │
│    "общий пул дискового пространства"        │
│    размер = сумма всех PV                   │
└─────────────────────────────────────────────┘
     ↓
┌─────────────────────────────────────────────┐
│    LV (Logical Volume)    LV (Logical Volume)│
│    "как обычный диск"      "еще один диск"   │
│    /dev/vg_name/lv_name                     │
└─────────────────────────────────────────────┘
     ↓
┌─────────────────────────────────────────────┐
│  Файловая система (ext4, xfs, и т.д.)       │
│  Монтируется в /mnt/...                     │
└─────────────────────────────────────────────┘
```

### 3.3 Преимущества LVM

| Возможность | Описание |
|-------------|----------|
| **Гибкость** | Изменение размера томов "на лету" (расширение/уменьшение) |
| **Снимки** | Мгновенные копии состояния ФС (snapshots) |
| **Объединение** | Один VG может включать несколько физических дисков |
| **Миграция** | Перемещение данных между PV без остановки |
| **Thin provisioning** | Выделение места по мере использования |
| **Striping** | Повышение производительности (аналог RAID 0) |
| **Mirroring** | Отказоустойчивость (аналог RAID 1) |

### 3.4 Snapshots (Снимки) — принцип работы

```
Исходный том (LV)               Снимок (Snapshot)
      │                               │
      ▼                               ▼
┌─────────────┐                 ┌─────────────┐
│  Данные A   │                 │   (пусто)   │
│  Данные B   │                 │   ссылается │
│  Данные C   │◄────────────────┤   на данные │
└─────────────┘                 │   исходного │
                               │   тома      │
                               └─────────────┘

После изменения данных B в исходном томе:

┌─────────────┐                 ┌─────────────┐
│  Данные A   │                 │  Данные B   │
│  Данные B'  │◄────────────────┤  (старые)   │
│  Данные C   │                 │   скопиро-  │
└─────────────┘                 │   ваны в    │
                               │   снимок    │
                               └─────────────┘

Результат:
- Снимок показывает состояние на момент создания
- Изменения в исходном томе не видны в снимке
- При изменении данных старая версия копируется в снимок
```

### 3.5 Команды LVM по уровням

| Уровень | Создание | Просмотр | Удаление | Изменение |
|---------|----------|----------|----------|-----------|
| **PV** | `pvcreate` | `pvdisplay`, `pvs` | `pvremove` | `pvresize` |
| **VG** | `vgcreate` | `vgdisplay`, `vgs` | `vgremove` | `vgextend`, `vgreduce` |
| **LV** | `lvcreate` | `lvdisplay`, `lvs` | `lvremove` | `lvextend`, `lvreduce` |
| **Snapshot** | `lvcreate -s` | `lvdisplay` | `lvremove` | - |

### 3.6 Размеры в LVM: экстенты (Extents)

**Extent** — минимальная единица выделения места в LVM.

```
PE (Physical Extent) = 4 МБ (по умолчанию)
LE (Logical Extent)  = 4 МБ (обычно = PE)

Все операции выделения места кратны размеру экстента.
```

**Указание размера при создании LV:**

| Параметр | Значение | Пример |
|----------|----------|--------|
| `-L 10G` | Абсолютный размер | 10 гигабайт |
| `-l 100` | Количество экстентов | 100 × 4 МБ = 400 МБ |
| `-l 50%VG` | 50% от размера VG | - |
| `-l 70%FREE` | 70% от свободного места в VG | - |

```
│     ┌──────────┼──────────┐                            │
│     ▼          ▼          ▼                            │
│ ┌────────┐ ┌────────┐ ┌────────┐                      │
│ │   LV   │ │   LV   │ │  LV    │                      │
│ │ mydata │ │ (своб) │ │(snap)  │                      │
│ │  70%   │ │  30%   │ │100%FREE│                      │

│ │  ext4      │      │  ext4      │                    │
│ │ /mnt/...   │      │ /mnt/...   │                    │
│ │   _base    │      │   _snap    │                    │
│ └────────────┘      └────────────┘                    │
└─────────────────────────────────────────────────────────┘
```

### 4.2 Почему такая последовательность?

| Шаг | Технология | Причина |
|-----|-----------|---------|
| 1 | iSCSI | Получить удаленные диски |
| 2 | RAID | Обеспечить отказоустойчивость на уровне дисков |
| 3 | LVM (PV) | Подготовить RAID-массив для LVM |
| 4 | LVM (VG) | Создать пул пространства |
| 5 | LVM (LV) | Создать гибкие тома для данных и снимков |

**Ключевой принцип:** LVM работает поверх блочных устройств (PV), которыми могут быть:
- Обычные разделы дисков
- RAID-массивы
- iSCSI-диски
- Любые другие блочные устройства

## 5. Основные команды для шпаргалки

### 5.1 iSCSI

```bash
# Установка
apt install open-iscsi

# Настройка имени
echo "InitiatorName=iqn.2025-04.ru.nstu.cs.vt:group.v9.initiator" > /etc/iscsi/initiatorname.iscsi

# Настройка CHAP в /etc/iscsi/iscsid.conf
node.session.auth.authmethod = CHAP
node.session.auth.username = login
node.session.auth.password = password

# Перезапуск сервисов
systemctl restart iscsid open-iscsi

# Discovery (поиск)
iscsiadm -m discovery -t sendtargets -p 172.16.6.15

# Login (подключение)
iscsiadm --mode node --targetname "iqn.2025-04.ru.nstu.cs.vt:group.v9" --login

# Просмотр сессий
iscsiadm -m session -o show

# Logout (отключение)
iscsiadm --mode node --targetname "iqn.2025-04.ru.nstu.cs.vt:group.v9" --logout

# Logout из всех сессий
iscsiadm -m node --logoutall=all
```

### 5.2 RAID (mdadm)

```bash
# Установка
apt install mdadm

# Создание RAID 1
mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sdb1 /dev/sdc1

# Просмотр статуса
cat /proc/mdstat
mdadm --detail /dev/md0
mdadm --detail --scan

# Сохранение конфигурации
mdadm --detail --scan >> /etc/mdadm/mdadm.conf
update-initramfs -u

# Остановка массива
mdadm --stop /dev/md0

# Удаление массива (очистка суперблоков)
mdadm --zero-superblock /dev/sdb1 /dev/sdc1
```

### 5.3 LVM

```bash
# PV (Physical Volume)
pvcreate /dev/md0           # создание
pvdisplay                   # детальная информация
pvs                         # краткая информация
pvremove /dev/md0           # удаление

# VG (Volume Group)
vgcreate vg-name /dev/md0   # создание
vgdisplay                   # детальная информация
vgs                         # краткая информация
vgextend vg-name /dev/sdX   # расширение
vgreduce vg-name /dev/sdX   # уменьшение
vgremove vg-name            # удаление

# LV (Logical Volume)
lvcreate -l 70%FREE -n lv-name vg-name    # создание
lvcreate -L 500M -n lv-name vg-name       # создание (абсолютный размер)
lvcreate -l 100%FREE -s -n snap-name vg-name/lv-name  # снимок
lvdisplay                   # детальная информация
lvs                         # краткая информация
lvextend -L +100M /dev/vg-name/lv-name    # расширение
lvreduce -L -100M /dev/vg-name/lv-name    # уменьшение
lvremove -f /dev/vg-name/lv-name          # удаление

# Файловая система
mkfs.ext4 /dev/vg-name/lv-name            # форматирование
mount /dev/vg-name/lv-name /mnt/point     # монтирование
umount /mnt/point                         # размонтирование
blkid /dev/vg-name/lv-name                # UUID и тип ФС
```

### 5.4 Работа с разделами (parted)

```bash
# Создание GPT таблицы
parted /dev/sdb --script mklabel gpt

# Создание раздела на весь диск
parted /dev/sdb --script mkpart primary 0% 100%

# Установка типа раздела "Linux raid"
parted /dev/sdb --script set 1 raid on

# Просмотр информации
parted /dev/sdb print
```

### 5.5 Диагностика

```bash
# Просмотр блочных устройств
lsblk
cat /proc/partitions
fdisk -l

# Просмотр логов
dmesg | tail
journalctl -xe
tail -f /var/log/syslog

# Проверка свободного места
df -h
vgdisplay
```

## 6. Типичные ошибки и их решение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `No records found` при login | Не выполнен discovery | Выполнить discovery сначала |
| Таргет не появился в discovery | Неправильный InitiatorName или CHAP | Проверить файлы, перезапустить сервисы |
| `dd if=/dev/random` зависает | Закончилась энтропия | Использовать `/dev/urandom` |
| `No such file or directory` в снимке | Снимок создан до появления файла | Удалить снимок и создать заново |
| `0 extents` при создании снимка | Нет свободного места в VG | Проверить `vgdisplay`, уменьшить LV |
| RAID не собирается при загрузке | Нет записи в mdadm.conf | Добавить запись и обновить initramfs |
