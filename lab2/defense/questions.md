## Ответы на вопросы (для защиты)

### 1. Что такое WMI?
WMI (Windows Management Instrumentation) — это инфраструктура управления в Windows, предоставляющая доступ к данным об операционной системе, оборудовании, приложениях и службах. Используется для мониторинга, автоматизации и удалённого управления.

### 2. Способы обращения к WMI из PowerShell
- `Get-WmiObject` (устаревший, но ещё работает)
- `Get-CimInstance` (современный, рекомендованный Microsoft)
- `[wmi]`, `[wmiclass]` ускорители типов
- WQL через `Get-WmiObject -Query` или `Get-CimInstance -Query`

### 3. WQL vs SQL
WQL (WMI Query Language) — подмножество SQL, но работает с WMI-классами, а не с таблицами БД. Не поддерживает `JOIN`, `ORDER BY` (только `WHERE` и простые условия).

### 4. Частые пространства имён WMI
- `root\cimv2` — системная информация (ОС, диски, процессы, сеть, службы)
- `root\default` — стандартные классы
- `root\wmi` — низкоуровневые данные (например, события питания)

### 5. Класс WMI, пример
Класс — это шаблон объекта. Пример: `Win32_OperatingSystem` — информация об ОС (имя, версия, серийный номер).

---

### 6. Использованные WMI-классы
- Имя компьютера: `Win32_ComputerSystem` (свойство `Name`)
- Пользователи: `Win32_UserAccount` (локальные учетные записи) или `Win32_LoggedOnUser` (активные сеансы) — задание требует «список пользователей», обычно подразумеваются локальные.
- Сетевые карты + MAC: `Win32_NetworkAdapterConfiguration` (где `IPEnabled = True`) или `Win32_NetworkAdapter`.

### 7. Имя файла
Формируется как `$env:COMPUTERNAME + ".rep"`. Расширение `.rep` — условное, по заданию (report).

### 8. Каталог сохранения
`C:\SysAdmLab2\`. Если каталог отсутствует, скрипт должен его создать: `New-Item -ItemType Directory -Force`.

### 9. Кодировка
Выбрана UTF-8 (или CP1251) для совместимости с русскими именами пользователей/программ. Реализация: `-Encoding UTF8` в `Out-File` или `Set-Content`.

### 10. Формат записей в файле
```
[users]
user1
user2

[network]
adapter1, MAC1
adapter2, MAC2
```
MAC-адрес обычно приводится к формату `XX-XX-XX-XX-XX-XX` (через дефис или двоеточие).

### 11. Обработка отсутствия сетевых карт
Выводится сообщение об отсутствии или просто пустой раздел `[network]` без строк.

### 12. Пример WQL-запроса в скрипте
```powershell
Get-CimInstance -Query "SELECT * FROM Win32_NetworkAdapterConfiguration WHERE IPEnabled = True"
```

### 13. Класс для автозагрузки (вариант 9)
`Win32_StartupCommand` — содержит команды и программы, запускаемые при старте системы (включая реестр, папки автозагрузки, задачи планировщика).

### 14. Выводимые свойства
Обычно:
- `Name` — название программы
- `Command` — исполняемая команда/путь
- `Location` — откуда загружается (реестр, папка Startup)
- `User` — для какого пользователя (если указано)

### 15. Добавление в файл
Скрипт добавляет новую секцию `[startup]` или дописывает строки в конец существующего файла. Например:
```
[startup]
Google Chrome, C:\Program Files\Google\Chrome\chrome.exe, HKLM\Software\Microsoft\Windows\CurrentVersion\Run
```

### 16. Автозагрузка текущего vs всех пользователей
- `Win32_StartupCommand` включает оба варианта.
- Можно отфильтровать по `Location` (например, `CurrentUser` или `Common Startup`).

### 17. Win32_StartupCommand ограничения
- Не показывает некоторые современные методы автозагрузки (например, запланированные задачи, службы, расширения оболочки).
- Может не включать программы, запускаемые через `RunOnce`.

### 18. Проверка корректности
1. Запустить скрипт от администратора.
2. Проверить создание каталога и файла.
3. Открыть файл в текстовом редакторе (Notepad++) — проверить кодировку.
4. Сравнить количество пользователей с `Get-LocalUser`, сетевых карт — с `ipconfig /all`, автозагрузки — с `msconfig` или Task Manager.
